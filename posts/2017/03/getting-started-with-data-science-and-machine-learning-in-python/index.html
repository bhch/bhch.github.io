Getting started with Data Science and Machine Learning in Python

Thu 23 March 2017   ·   Programming

It's been a long time since I first thought about learning Data Science and Machine Learning. If you, too, want to get started with ML, let's embark on this
journey together.

In this post I will share how I plan to get started with ML. This isn't supposed to be a guide, per se. I'm also a beginner in this, you know. Feel free to leave suggestions in comments below.

Do you really need Math?

In case you're wondering if you really need to know the ML related Math that bad before doing some actual ML stuff, then no, you don't. I believe in learn as you go approach. Don't get me wrong, I like Math (espesially Functions and Limits). But I want to get started quickly and will learn the necessary Math when I'll need it. Here's a post on R-bloggers — The real prerequisite for machine learning isn't math, it's data analysis and here's another on Cross Validated — Is a strong background in maths a total requisite for ML? that you should read. Both of these posts present pretty solid discussions why Math isn't an outright necessity for ML.

Installing packages

Before installing anything, it would be a better if you create a virtualenv just for Machine Learning. And then install the following packages in that virtualenv.

  1. IPython
    The de facto IDE for scientific computing.
  2. Numpy
    For array computing, also, numpy arrays are awesome.
  3. Pandas
    For data manipulation.
  4. Matplotlib
    For data visualization i.e. plotting graphs, etc.
  5. Scipy
    For scientific computing. Also, many other packages listed here require this.
  6. Scikit-learn
    For Machine Learning.
  7. Scrapy
    For web scraping.
  8. BeautifulSoup
    For web scraping and parsing,
  9. Requests
    It's like urllib but far better.

To install, just copy the following command and run it in your terminal:

$ pip install numpy scipy matplotlib ipython jupyter pandas scikit-learn Scrapy beautifulsoup4 requests

Now what?

First of all, we'll need to read through the Gettings started docs of the above libraries. However, I plan to do things in the following order:

  1. Web scraping (request, beautifulsoup, scrapy).
  2. Data structuring and manipulation (pandas).
  3. Data visualisation (numpy, matplotlib).
  4. Machine learning (scikit-learn).

I think the above order is pretty good because learning curve isn't very steep right from the start. I've had some experience with web scraping before, so it should be relatively easier to start with that instead of directly getting into ML. Think of it as paving your way into ML.

In my next blog post, I plan to share the first web scraping project that I will do.
